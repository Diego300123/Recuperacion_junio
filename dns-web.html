1. Objetivo

  • Configurar un servidor DNS maestro utilizando BIND9 para el dominio zaidin.test, incluyendo varios registros típicos como A, CNAME, MX, etc. Comprobar la correcta resolución de nombres mediante herramientas como dig o nslookup.
  • Configurar un servidor web seguro con NGINX.

2. Configuración cliente de DNS

Configuraremos una máquina virtual cliente para probar el DNS.

  • Su dirección será 192.168.57.61
  • Instalaremos las herramientas bind9-utils que contiene dig y nslookup.

3. Instalación del servicio DNS

Crea una máquina virtual para el servidor dns. Instala el paquete bind9 y asegúrate de que el servicio se inicie correctamente y se habilite al arranque.

3.1. Configuración de la zona directa

Configura la zona directa zaidin.test.

Define al menos los siguientes registros:

  • Registro SOA y NS (tú servidor dns).
  • Registro A del servidor principal (192.168.57.10).
  • www - registro A del servidor web (192.168.57.20).
  • mail - con dirección 192.168.57.30 y prioridad 20.
  • client - con dirección 192.168.57.61.

3.2. Configuración de la zona inversa

Configura la zona inversa correspondiente a 192.168.57.X.

3.3. Comprobación del servicio

Comprueba el servicio (mediante dig o nslookup) desde la máquina virtual cliente. Realiza capturas del funcionamiento de la zona directa y la zona inversa.

4. Configura un servidor web seguro

Configurar un servidor web NGINX con las siguientes características:

  • IP del servidor: 192.168.57.20
  • Nombre de dominio completo (FQDN): www.zaidin.test
  • El alumno deberá configurar el acceso HTTPS utilizando un certificado autofirmado creado expresamente para este dominio.
  • El subdirectorio /app/ deberá estar protegido mediante autenticación básica con usuario y contraseña.

4.1. Configuración de un servidor NGINX seguro

  1. Instalar y configurar el servidor NGINX.
  2. Crear una página de prueba HTML accesible desde `https://www.zaidin.test/`.
  3. Generar un certificado SSL autofirmado y configurar NGINX para utilizarlo.
  4. Configurar un subdirectorio /app/ dentro del sitio web que requiera autenticación mediante usuario y contraseña.
    • El usuario se llamará alumno.
    • La contraseña será 123
  5. Crear una segunda página de prueba HTML accesible solo desde `https://www.zaidin.test/privado/`.

4.2. Comprobaciones

Configuremos en el ordenador anfitrión como servidor DNS por defecto el servidor 192.168.57.10 (nuestro servidor DNS).

El alumno deberá comprobar y realizar una captura de:

  • El sitio principal es accesible a través de HTTPS en https://www.zaidin.test/.
  • La conexión utiliza el certificado configurado.
  • El subdirectorio `/privado/` solicita usuario y contraseña para poder acceder.
  • La página de prueba en `/privado/` se muestra correctamente tras autenticarse.

5. Entrega

El alumno deberá entregar un archivo comprimido (.zip) que contenga:

  • Todos los ficheros modificados o creados
  • Un documento con las capturas que demuestran el funcionamiento de todos los requirimientos solicitados.

6. Evaluación

Se evaluará:

  • Corrección en la configuración del FQDN y resolución local.
  • Funcionamiento del sitio en HTTPS.
  • Correcta protección del subdirectorio mediante autenticación.
  • Claridad y organización de los ficheros entregados.
